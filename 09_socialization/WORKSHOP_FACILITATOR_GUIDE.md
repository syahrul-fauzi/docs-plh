# Panduan Fasilitator Workshop (Workshop Facilitator Guide)

Dokumen ini memberikan panduan terstruktur bagi fasilitator untuk menjalankan
workshop sosialisasi arsitektur dokumentasi enterprise Lawyers Hub.

## 1. Persiapan Sebelum Workshop

- **Materi**: Pastikan `WORKSHOP_ENGINEERING.md` dan `WORKSHOP_LEGAL.md` sudah
  diperbarui.
- **Lingkungan**: Siapkan akses ke repository GitHub Lawyers Hub.
- **Peserta**: Kirimkan undangan minimal 2 hari sebelumnya dengan tautan ke
  `SOCIALIZATION_ROADMAP.md`.

## 2. Struktur Sesi Workshop

Setiap sesi dirancang untuk menjadi interaktif dengan alur berikut:

### A. Pengantar (15-20 Menit)

- Jelaskan **"The Why"**: Mengapa kita melakukan restrukturisasi? (Fragmentasi,
  kepatuhan UU PDP, skalabilitas).
- Tunjukkan struktur 9 pilar dan manfaatnya bagi masing-masing tim.

### B. Demonstrasi Praktis (30-40 Menit)

- **Engineering**: Tunjukkan cara kerja `markdownlint` di lokal dan CI/CD.
  Demonstrasikan pembuatan API Docs otomatis.
- **Legal**: Tunjukkan cara mengisi `rule_prd_template.md` dan bagaimana itu
  dipetakan ke sistem.

### C. Diskusi Interaktif & Tanya Jawab (20-30 Menit)

- Gunakan skenario nyata: "Bagaimana jika ada perubahan undang-undang baru?"
  atau "Di mana saya menaruh hasil audit mingguan?".
- Dorong peserta untuk mencoba mengedit dokumen menggunakan template.

### D. Pengumpulan Feedback (10 Menit)

- Minta peserta mengisi formulir feedback di `FEEDBACK_FORM.md`.

## 3. Tips untuk Fasilitator

- **Suasana**: Ciptakan suasana yang nyaman bagi peserta untuk bertanya,
  terutama bagi tim non-teknis (Legal).
- **Proaktif**: Jika peserta pasif, tunjukkan contoh dokumen lama yang
  membingungkan dan bandingkan dengan format baru.
- **Catatan**: Dokumentasikan pertanyaan kunci yang belum terjawab untuk
  ditindaklanjuti.
